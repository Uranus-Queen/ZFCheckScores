## 重构 ZFCheckScores 主流程 + 修复「推送非本学期成绩」

### 根因
`main.py` 调用 `get_grade()` 无参 → `student_client.get_grade(year=0, term=0)` 请求**全部学年**成绩。
`get_grade.py` 把全部学期成绩按 `submission_time` 取前 8 条做 MD5 比对，学校补录/更正**任何历史学期**成绩都会触发误判为「成绩已更新」并推送。
接口本身支持按学期查询（term=1→第一学期 xqm=3，term=2→第二学期 xqm=12），此前从未传入。

### 改动
- 新增 `scripts/semester.py`：`calendar_semester`(日期兜底) / `selected_courses_semester`(已选课程取最大学年学期) / `resolve_current_semester`(优先已选课程，失败回退日期) / `semester_label`。
- `scripts/get_grade.py`：`get_grade` 增加 `semester=None` 参数，传给 `student_client.get_grade(year, term)`；None 时仍抓全学期（GPA 总览用）。
- `main.py`：模块化重构（load_config / is_first_run / snapshot_grade_file / fetch_and_store_grade / build_* / main），接入 `resolve_current_semester` 把**抓取与变更检测**限定在当前学期；修复原 `run_count == 1` 空操作等潜在问题。

### 关键决策
- 学期判定采用「已选课程优先 + 日期兜底」：用户确认补考/重修成绩登录在当前学期分栏，按学期过滤不会漏。
- **GPA 保持累计（未随学期过滤）**：`get_user_info` 仍不传 semester，行为不变、零回归；仅成绩展示与变更检测按学期过滤。
